There is **no way** to skip creating a Google Cloud project — Google's OAuth 2.0 policy is a hard security rule. Every OAuth client is bound to specific JavaScript origins. No shared key exists. Here's the fastest setup (under 3 minutes): **Step 1** — [Create project](https://console.cloud.google.com/projectcreate) (any name) **Step 2** — [Enable YouTube Data API v3](https://console.cloud.google.com/apis/library/youtube.googleapis.com) **Step 3** — [Create Credentials](https://console.cloud.googleapis.com/apis/credentials) → OAuth client → Web application **Step 4** — Copy your Client ID from the credentials page, paste it in the input field, click Save **Step 5** — Serve locally: `python -m http.server 8000` in the folder, then open `http://localhost:8000/yourfile.html` **Step 6** — Click Sign in with Google That's it. After this one-time setup, it's fully automatic — sign in once, then click Start and it runs forever until you click Stop. The Client ID saves to localStorage so you never need to do it again even after page reload. **Why no workaround exists:** Google's OAuth requires every app to register its origin. A "shared" key would be a security vulnerability — any website could impersonate your app. Google blocks this on purpose. The Client ID is literally just an identifier — it doesn't contain any secrets — it just tells Google "this origin is allowed".
